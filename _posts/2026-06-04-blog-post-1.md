논리회로
------

02. 동기식 카운터

**카운터란?**  
숫자를 세는 회로, 클록 신호가 들어올 때마다 숫자가 하나씩 증가

**동기식 카운터란?**  
모두 동시에 움직인다는 뜻. 즉, 카운터를 구성하는 모든 플립플롭이 같은 클록 신호를 동시에 받음.

**장점**  
시간 지연이 거의 없음(동기식 카운터를 사용하는 핵심 이유)
비동기식 카운터는 각 플립플롭이 순서대로 바뀜. 그래서 아주 짧은 순간 이상한 값이 잠깐 나타날 수 있음
동기식 카운터는 모든 플립플롭이 동시에 바뀌므로 바로 변경가능.
따라서 1. 속도가 빠르고 2. 오동작이 적으며 3. 고속 회로에 적합하다.

**카운터가 필요한 이유?**  
여러 가지 기능이 있으면 서로를 구분하는 것이 필요함.
즉, 각각의 기능을 구분하기 위해 카운터를 이용.

03. 기타 카운터
**1. 링 카운터**   
임의의 시간에 한 개의 플립플롭만 논리 1이 되고 나머지 플립플롭은 논리 0이 되는 카운터
쉬운 설명 : 
1 하나가 플립플롭들을 따라 원형으로 계속 이동하는 카운터

- 사용하는 이유?
  - 각 상태가 하나씩만 켜지기 때문에 순서대로 동작해야 하는 곳에 사용함
- n개의 플립플롭이면 상태 수는 n개

  - <img width="622" height="211" alt="image" src="https://github.com/user-attachments/assets/5858056d-60aa-4867-876b-4a17a758495d" />
  - 0부터 15까지 카운터가 되면 카운터가 하나 만들어짐 -> 뭔말이지 이게

쉬운 설명 :
- IC는 집적회로(Integrated Circuit)의 약자로 카운터를 만들기 위해 필요한 플립플롭과 논리게이트들을 하나의 칩 안에 넣어놓은 것.
- 비유 -> 레고 자동차
  - 직접 레고로 자동차를 만들면 모든 부품을 하나하나 조립해야 함.
  - IC카운터 사용 : 이미 완성된 자동차를 사는 것
  - 필요한 기능이 칩 안에 다 들어 있어서 전원,클록 연결만 해주면 바로 카운터로 사용가능

**IC카운터를 사용하는 이유**  
- 직접 만들면 플립플롭 + 논리게이트 + 배선이 많이 필요함
- IC카운터는 IC카운터 칩 하나만 사용하면됨
- 그래서 회로가 간단해지고 공간이 적게 들며, 오류가 적고 제작이 쉽다.

**IC카운터 핵심 정리**
- 카운터 기능을 하나의 집적회로에 넣은 칩
- 클록 펄스를 자동으로 센다.
- 플립플롭과 논리회로가 내부에 포함되어 있다.
